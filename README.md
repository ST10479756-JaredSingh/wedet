## 2. New Entries for Part 2  

### üñå General Page Styling  
- Uses a clean sans-serif font stack (Segoe UI, Arial) for modern readability.  
- Background color: light beige creates a soft, neutral canvas.  
- Text color: dark gray ensures high contrast and readability.  
- Removes default browser spacing for consistent layout control.  

**Purpose:** Provides a neutral, readable base that makes headers, cards, and content stand out.  
*(W3Schools, 2025)*  

---

### üé® Header  
- Gradient background: Horizontal colour gradient creates a visual focal point.  
- Text centered and white: Makes site title or heading clearly visible.  
- Padding: `1rem` to ensure breathing space around header content.  

**Purpose:** Acts as a visual introduction to the site, making it attractive and attention-grabbing.  
*(W3Schools, 2025)*  

---

### üîó Navigation Bar  
- Background: Dark green, complementing the header gradient.  
- Flex layout: Horizontal row, center aligned, evenly spaced links.  
- Hover effects: Links slightly rise and underline, adding interactive feedback.  
- Responsive adjustments: Links expand and wrap on smaller screens, making the nav touch-friendly and readable.  

**Purpose:** Provides intuitive, visually appealing navigation while adapting to different screen sizes.  

---

### üì¶ Main Content Container  
- Max-width: 75% on desktop to prevent content from stretching too wide.  
- Centered container: `margin: auto` ensures focus is on content.  
- Padding and border radius: Softens the edges and separates content visually.  

**Purpose:** Holds the main site content in a readable, aesthetically pleasing block.  

---

### üÉè Cards  
- Background: Soft beige for contrast against the white container.  
- Border-radius & padding: Adds visual softness and spacing.  
- Flex layout inside cards: Stacks content vertically, centers items.  
- Width: 25% on desktop while having a responsive width on smaller screens.  
- Hover effects: Slight lift which adds interactivity and depth.  
- Images: Rounded, `object-fit` ensures consistent aspect ratios.  

**Purpose:** Highlight content (values, team members) in individual, attention-grabbing blocks that feel dynamic.  
*(W3Schools, 2025)*  

---

### üìù Footer  
- Matches navigation bar styling for visual consistency.  
- White text ensures readability.  
- Links underlined to indicate clickable content.  

**Purpose:** Provides final site navigation and contact info while staying visually consistent with the theme.  

---

### üì± Responsiveness  
- Container width: Increases on smaller screens for better use of space.  
- Navigation links: Wrap and expand to full width, ensuring touch usability.  
- Cards: Expand width and stack vertically while images scale down.  
- Grid layouts: Switch from horizontal flex to vertical stacking on narrow screens.  

**Purpose:** Ensures the site looks good and functions well on all devices, from desktops to mobile phones.  
*(W3Schools, 2025)*

## 3. References Section  

### 3.1 Purpose  
This project was created as part of my learning and practice in CSS and web development. To improve my skills and gain better understanding of CSS concepts, I used several online resources and tutorials.  

### 3.2 Resources Used  
1. [W3Schools CSS Tutorial](https://www.w3schools.com/Css/)  
   - Beginner-friendly tutorial covering CSS basics, selectors, properties, and practical examples.  
   - Provided hands-on examples to practice coding directly in the browser.  

2. [FreeCodeCamp](https://www.freecodecamp.org/)  
   - Comprehensive course that covers CSS fundamentals, including styling, layouts, and responsive design.  
   - Helped me understand how to structure and style web pages effectively.  

3. [FrontendLead Handbook](https://frontendlead.com/handbook/learn-css-for-beginners)  
   - Step-by-step guide for beginners to learn CSS and apply it in real projects.  
   - Offered practical tips and exercises to solidify my understanding.  

4. [Canva](https://www.canva.com/)  
   - Used Canva to explore design layouts and colour schemes.  
   - Provided visual guidance and inspiration for implementing CSS styles in my project.  

5. [AdvTech Online SharePoint](https://advtechonline.sharepoint.com/)  
   - Helped me build the foundation of my design and gave ideas for attributes and data to use in a working stylesheet (eg. Home.css).
